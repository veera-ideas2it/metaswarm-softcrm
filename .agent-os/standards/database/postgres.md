# Standard: PostgreSQL Database
# Applies to: all database models, migrations, queries

## Version
- PostgreSQL 15

## Universal Column Rules
Every table MUST have these columns:
```sql
id          UUID PRIMARY KEY DEFAULT uuid_generate_v4()
created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
deleted_at  TIMESTAMPTZ NULL  -- soft delete, NULL means active
```

## Soft Deletes
- Never hard DELETE rows — set deleted_at = NOW()
- All queries must filter WHERE deleted_at IS NULL
- SQLAlchemy: use a query filter in the base service class

## Indexes
Always create indexes on:
- All foreign key columns
- Columns used in WHERE filters (stage, owner_id, email)
- Columns used in ORDER BY (created_at on most tables)
- Full-text search columns (name, email) — use GIN index

Naming: idx_{table}_{column(s)}
Example: idx_deals_owner_id, idx_deals_stage, idx_contacts_company_id

## Migrations (Alembic)
- Every schema change = one Alembic migration file
- Never ALTER tables manually in production
- Migration filename: auto-generated by Alembic (revision id)
- Migration message: descriptive verb_noun (create_deals_table, add_probability_to_deals)
- Always write both upgrade() and downgrade()
- Test downgrade before merging

## Foreign Keys
- Always define explicit FK constraints
- ON DELETE behavior: RESTRICT by default (never silently cascade deletes)
- Exception: stage_history → deals: ON DELETE CASCADE (log dies with deal)

## Data Types
- IDs: UUID (not integer — avoids enumeration attacks)
- Money/amounts: NUMERIC(12, 2) — never FLOAT for currency
- Text fields: VARCHAR with appropriate max length, or TEXT for unlimited
- Dates: TIMESTAMPTZ (always timezone-aware), DATE for date-only fields
- Enums: PostgreSQL native ENUM type, also define in SQLAlchemy

## Naming
- Tables: snake_case, plural (deals, stage_histories, not deal or StageHistory)
- Columns: snake_case (first_name, created_at)
- FKs: {singular_table}_id (company_id, owner_id, contact_id)

## Connection
- Use async connection pool (asyncpg)
- Pool size: 5 min, 20 max
- Connection string via DATABASE_URL env var only
